巴哈姆特自動化 使用教學

第一步：建立 GitHub 帳號

如果你還沒有 GitHub 帳號，請先建立一個帳號。

第二步：前往核心程式的專案

前往 Bahamut Automation 專案。
如果你願意，請點擊右上角「 Star 」按鈕，給專案一顆閃亮的 Star。
然後，請點擊右上角「 Watch 」按鈕，可以收到專案更新的訊息。
接著務必把下方的 README.md 文件內容仔細看過。

第三步：Fork Template 專案

前往 Template 專案。
請點擊右上角「 Fork 」按鈕，複製一份 Template 專案。

接著點擊「 Settings 」按鈕，在下方「 Features 」區塊中找到 「 Issues 」，並把它打勾。

再來，點擊左方選單的「 Secrets 」，點擊「 New repository secret 」按鈕，在「 Name 」欄位輸入「 secrets 」，其值稍後輸入，不要關閉此頁。

第四步：建立設定檔

前往 設定檔產生器，輸入你的巴哈帳號及密碼。
如果有開啟兩步驟驗證，記得輸入兩步驟驗證種子碼（不是那六位數字，應該是一串大寫英文）。

如果要啟用 GitHub Issue Report 請依照以下步驟：
  1. 點擊「 產生 」按鈕
  2. 點擊該頁右上角「 Generate new token 」按鈕
  3. 在「 Note 」欄位中輸入任何字（提醒你這個 Token 是幹嘛用的）
  4. 調整「 Expiration 」至「 No expiration 」
  5. 勾選下方「 workflow 」
  6. 至最底部點擊「 Generate token 」
  7. 複製像「 ghp_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX 」的 Token 到對應的產生器欄位

如果要啟用 Telegram Report 請依照以下步驟：
  1. 點擊「 機器人 」按鈕
  2. 開啟 Telegram 應用程式並與機器人「 Automia 」開始對話
  3. 傳送「 ID 」給機器人「 Automia 」
  4. 複製像「 1010101010 」的通道 ID 到對應的產生器欄位

如果要啟用 Discord Report 請依照以下步驟：
  1. 建立一個自己的 Discord 伺服器，在你想要發送的文字頻道按 齒輪 設定
  2. 點擊 整合 -> Webhook -> 新 Webhook
  3. 點擊 複製 Webhook 網址 並貼到到對應的產生器欄位

如果要啟用 Line Notify Report 請依照以下步驟：
  1. 進入 Line Notify
  2. 點擊右上方登入 Line 帳號
  3. 點擊右上方 自己的 ID -> 個人頁面 -> 發行權杖
  4. 填入名稱(會顯示在訊息上)，並選擇要接收訊息的聊天室或群組，建議選擇 透過1對1聊天接收LINE Notify的通知，若選擇群組的話，需將 Line Notify 官方帳號加入群組後才收的到訊息
  5. 按下發行
  6. 點擊 複製，並將權杖代碼貼到對應的產生器欄位

按下「 複製 」按鈕複製設定檔，並貼回剛剛第三步的「 Value 」欄位，按下「 Add secret 」按鈕新增 Secret。

第五步：啟用 GitHub Action

前往你的專案的「 Actions 」分頁。
如果你看到像「啟用 (Enable) 」的綠色按鈕，把它按下去。
選擇左方選單的「 自動化 」，然後點擊右側「 Run workflow 」按鈕。 不久後，應該就會執行第一次自動化，可以點進去看看發生什麼事。

第六步：遇到問題

前往 Bahamut Automation 專案的 Discussions 頁面。
找找有沒有人跟你遇到一樣的問題，如果沒有，請在這裡建立一個 Discussion (請用 Q&A 類別)，並盡可能詳細的描述你的問題。
